# Incidents

Post-incident records for Babbla: what happened, what held, what failed, the root
cause, and the remediation. One file per incident, named `YYYY-MM-DD-short-slug.md`.

These are blameless engineering records — the goal is an honest account and concrete
action items, not attribution. Where an incident exposes an overstated guarantee, the
fix includes correcting the relevant docs (README / ADRs) so the written contract
matches verified behavior.
